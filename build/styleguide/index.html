Typography

Typeface

The brettlguide.at typeface is the sans serif font Lato which was designed in sum­mer 2010 by Warsaw-​​based designer Łukasz Dziedzic (“Lato” means “Sum­mer” in Pol­ish). In Decem­ber 2010 the Lato fam­ily was pub­lished under the open-​​source Open Font License by his foundry tyPoland, with sup­port from Google.

For test purposes you can also use Google Fonts code to embed the font in your project

<style>
@import url('https://fonts.googleapis.com/css?family=Lato');
</style>

Use the following CSS rules to specify Lato font in your CSS-file

font-family: 'Lato', sans-serif;

Try to avoid embedding lato font through Google fonts and use locally hosted fonts located in ./assets/fonts/ instead.

Methodology

Code style rules

Block indentation

Each time a new block or block-like construct is opened, the indent increases by two spaces. When the block ends, the indent returns to the previous indent level. The indent level applies to both code and comments throughout the block.

Color Variables

Each color that is used more than once should be placed in a variable in color.css.

Preprocessor

As a preprocessor we use postcss. PostCSS is a tool for transforming styles with JS plugins. These plugins can lint your CSS, support variables and mixins, transpile future CSS syntax, inline images, and more.

Postcss plugins we use

autoprefixer
to add vendor prefixes, using data from Can I Use

postcss custom properties
to transform CSS Custom Properties (CSS variables) syntax into a static representation

postcss import
to consume local files, node modules or web_modules

postcss style-guide
to create a styleguide from YAML-structured CSS-comments.

postcss nested
to unwrap nested rules like how Sass does it.

Class naming conventions

To make sure our CSS is structured we use the BEM naming convention. The BEM naming convention distinguishes between blocks, elements and modifiers.

Blocks are independet and named with a short class name

<div class="site-search">...</div>

Elements are parts of blocks and can't be used by themselves

<div class="site-search">
    <input type="text" class="site-search__field">
    <input type="Submit" value ="Search" class="site-search__button">
</div>

Modifiers change the look of blocks or elements

<div class="site-search site-search--full">
    <input type="text" class="site-search__field">
    <input type="Submit" value ="Search" class="site-search__button">
</div>
